## Project Overview

Speech Emotion Recognition (SER) research project for a BIT graduation paper. Compares baseline and attention-based CNN architectures on the RAVDESS dataset (8 emotion classes, 24 actors). Evaluation is subject-independent via Group K-Fold cross-validation split by actor ID.

## Commands

**Setup**:
```bash
conda create -n grad_paper_ser python=3.9
pip install -r requirements.txt
```

**Smoke test** (single epoch, verifies pipeline):
```bash
python src/main.py model=cnn_baseline
```

**Full training** (5-fold cross-validation):
```bash
python src/train.py model=cnn_baseline
python src/train.py model=cnn_temporal_attention
python src/train.py model=cnn_channel_attention
python src/train.py model=cnn_spatial_attention
python src/train.py model=mobilenet_v3_small
python src/train.py model=efficientnet_lite0
```

**Inference**:
```bash
python src/infer.py audio_path="path/to/audio.wav"
```

**Mel-spectrogram visualization**:
```bash
python src/visualize_mel.py
```

**View experiment results**:
```bash
mlflow ui
```

Hydra auto-generates output directories: `outputs/{YYYY-MM-DD}/{HH-MM-SS}/` containing weights, learning curves, confusion matrix, class accuracy chart, t-SNE visualization.

## Architecture

### Data Pipeline (`src/data/`)
Audio files → Resample to 16kHz mono → Log-Mel Spectrogram (128 mel bins, n_fft=1024, hop=512) → Bicubic resize to fixed 128×512 → Normalize. Output tensor shape: `(1, 128, 512)`.

`RavdessDataset` parses filenames in format `03-01-01-01-01-01-01.wav`: 3rd segment = emotion label (1-8), 7th segment = actor ID. Returns `(audio_path, emotion_label, actor_id)`.

`collate_dynamic_padding` handles variable-length spectrograms within a batch.

### Model Registry (`src/utils/registry.py`)
Central decorator-based registry. Register new models with `@register_model("name")`. Config-driven model selection: `python src/train.py model=<name>` instantiates from `src/configs/model/<name>.yaml`.

### Models (`src/models/`)
- `base.py` — `cnn_baseline`: 4 VGG-style conv blocks [64,128,256,512] + optional spatial attention. AdaptiveAvgPool(4,4) → Linear → 8 classes.
- `temporal_attention.py` — `cnn_temporal_attention`: Baseline + temporal attention that compresses frequency dim, scores each time step via 1D conv, and takes weighted sum. Focuses on emotional bursts.
- `channel_attention.py` — `cnn_channel_attention`: Baseline + SE-blocks (Squeeze-and-Excitation) after each conv block. Learns per-channel importance to enhance discriminative frequency patterns.
- `spatial_attention.py` — `cnn_spatial_attention`: Baseline + CBAM spatial attention maps. Highlights critical (frequency, time) positions in spectrograms.
- `mobilenetv3.py` — `mobilenet_v3_small`: Lightweight architecture, first conv modified for 1 input channel, classifier replaced for 8 classes.
- `efficientnet_lite.py` — `efficientnet_lite0`: EfficientNet-Lite variant with same 1-channel / 8-class modifications.

### Configuration (`src/configs/`)
Managed by Hydra. Key files:
- `config.yaml`: seed=42, batch_size=32, k_folds=5, early_stopping patience=10, device="auto"
- `data/default.yaml`: sample_rate, n_mels, n_fft, hop_length, resize dimensions
- `model/*.yaml`: One file per model architecture with its hyperparameters

### Training (`src/train.py`)
Group K-Fold (k=5) splits by actor ID. Each fold: fresh model init, Adam optimizer, CrossEntropyLoss, early stopping. Per-fold outputs: learning curves, best weights. After all folds: global confusion matrix, class-wise accuracy bar chart, t-SNE visualization. MLflow logs hyperparams, per-fold metrics, and artifacts.

## Code Conventions

- Feature extraction functions follow `extract_<method>` naming (e.g., `extract_fft`, `extract_mfccs`)
- Use type hints for all function parameters and return values with modern Python 3.10+ syntax: `list[str]`, `str | None`
- Use f-strings, `pathlib` for paths, and context managers for resource management
- PEP 8 import order: stdlib → third-party → local; no wildcard imports
- New models must be registered with `@register_model` and have a corresponding config YAML
- External library additions must be reflected in `requirements.txt`

### **2. “语音情绪识别” vs “语音情绪识别技术” 의 차이**

| **표현** | **의미** | **사용 상황** |
| --- | --- | --- |
| **语音情绪识别** | **연구 분야/과제 자체** | 연구 분야를 소개할 때, 연구 동향을 말할 때 |
| **语音情绪识别技术** | **그 과제를 해결하는 구체적인 방법/시스템/능력** | 기술의 성능, 적용 가능성, 공학적 가치를 말할 때 |

**쉽게 말하면**:

- “语音情绪识别” = “이 분야는 중요하다” (연구 영역)
- “语音情绪识别技术” = “이 분야에서 개발된 기술/시스템은 실용적이다” (구체적 수단)

1. **"겸손한 어법" (Defensive Writing)**
    - `~이다`, `가장 ~하다`, `이미 해결했다`와 같은 단정적 표현을 피하세요.
    - 대신 `~중 하나로 간주된다(之一)`, `~하는 경향을 보이기 시작했다(开始向着...发展)`, `중요한 역할을 한다` 등으로 서술하여 논리적 공격을 피해야 합니다.
2. **"용어의 일당백" (One Term Policy)**
    - 똑같은 대상을 부를 때 여러 단어를 섞어 쓰지 마세요. (예: 시스템, 플랫폼, 도구 -> `人机交互系统` 하나로 통일)
    - 지루해 보일 정도로 한 단어만 고집하는 것이 학술적 일관성을 증명합니다.
    - 人机交互系统
    - 媒介
    - 语音
3. **"데이터와 알고리즘의 분리" (Media vs. Method)**
    - 얼굴, 목소리는 '방법'이 아니라 정보를 실어 나르는 **'매개체(媒介/载체)'**입니다.
    - 알고리즘이나 기술적 수단만 `方法`이라고 부르세요.
4. **"대조군의 명확화" (Text vs. Acoustic)**
    - 단순히 '언어적/비언어적'이라고 뭉뚱그리지 마세요.
    - *문자(文字语义)**와 **비문자(非文字特征)**라는 대조군을 사용하여 연구의 분류 체계를 선명하게 보여주어야 합니다.
5. **"서술의 한자 선택" (Precision in Characters)**
    - `制订`(안을 짜다)보다 `制定`(확정하여 결정하다)처럼 동작이 완료되고 확정된 느낌의 한자를 선택하여 연구의 권위를 높이세
6. 문장의 주어 반드시 배치
    - 모든 문장에는 적합한 적절한 주어를 배치하여서, 설명을 직관적이고 더 이해할 수 있게 문장 구조를 탄탄하게 해야 한다
7. **"현상"보다 "기제(Mechanism)"를 써라**:
    - "A 기술이 많이 쓰인다"라고 쓰지 말고, "A 기술은 **~한 원리로 ~를 해결하기 때문에** 최근 주요 연구 방향으로 자리 잡았다"라고 쓰세요. **detail.md** 워크플로우의 '기술 서술 패턴'과 일맥상통합니다.
8. **형용사적 나열보다 동사적 작용을 써라**:
    - "중요하다", "활발하다" 같은 형용사적 서술보다는, "**주목한다(关注)**", "**강화한다(增强)**", "**운반한다(承载)**"와 같이 기술이 데이터에 가하는 구체적인 '작용'을 서술하세요.
9. **학술적 무게 중심의 이동을 묘사하라**:
    - 고정된 상태가 아니라, 학계의 관심사가 어디에서 어디로 **'이동(转向)'**하고 있는지 서술함으로써 사용자님의 연구가 그 '최신 흐름'의 정점에 있음을 간접적으로 증명하세요. (직접 "내 연구가 최신이다"라고 말하지 않는 것, 즉 Zero-Absolutism의 응용입니다.)
10. **'학부생(本科)' 관련 수식어 전면 삭제**:
    - "학부 단계에서 배우는...", "학부생으로서 시도하는..." 같은 표현은 모두 삭제하세요. 대신 `本研究`(본 연구) 또는 `本文`(본문)이라는 표현을 사용해 연구 그 자체의 가치로 승부해야 합니다.
11. **주어의 구체화 (`其` 자제)**:
    - 문장 시작에 `其`를 쓰는 대신, `本 연구의 ~는`, `해당 모델의 ~는`과 같이 명사를 명확히 사용하세요. 이는 문장의 직관성을 높일 뿐만 아니라 심사위원들에게 필자가 논리를 확실히 장악하고 있다는 인상을 줍니다.
12. **'규모'보다 '정교함' 강조**:
    - "거창한 모델이 아니다"라는 부정적 방어보다는, "**실제 데이터(实际数据)**와 **구체적 구조(具体结构)** 내에서의 **적절한 배치(恰当部署)**"와 같이 작업의 '정밀도'를 강조하는 단어를 선택하세요
13. **사자성어 및 관용구 사용 자제**:
    - 중국어 논문을 쓸 때 멋있어 보이려고 사자성어를 쓰는 경우가 많지만, 이공계 논문에서는 오히려 독이 될 수 있습니다. 뜻이 명확한 **2음절 또는 4음절의 공학 전문 용어**를 쓰세요.

- 도입 단락은 **해당 장에서 다룰 내용의 목록**을 명시하여 독자가 구조를 파악하게 함
- 기술 장(제2장)은 이론 → 모델 → 평가지표 순서로 전개

### **2. 기술 항목 서술 구조 (定义→机制→应用 패턴)**
- 각 소절은 반드시: **①정의** → **②작동 원리/공식** → **③장점** → **④한계** 순서
- "被提出" 수동태로 해당 기술의 역사적 등장 서술 (예: "多层感知机...被提出")
- 기술 개념은 최초 등장 시 **영문명 + 중문명 + 약어**를 함께 제시 (예: 循环神经网络（Recurrent Neural Network，RNN）)

### **3. 항목 번호 형식**
- 기술 항목의 하위 목록: "1. Sigmoid函数", "2. ReLU函数" 형식 (볼드+번호)
- 또는 `\textbf{(1) 가성주의력}` 형식 사용
- 항목 나열 시 반드시 **독립 단락**으로 처리 (불릿 기호 사용 금지)

### **4. 수식 번호 및 인용**
- 모든 수식에 (2.1), (2.2) 형식 번호 필수
- 논문 내 인용: [1], [2] 숫자 형식 (LaTeX에서는 \cite{} 이용)
- 인용 위치: 서술 문장 끝, 마침표 앞

### **6. 문체적 특징**
- 주어는 명확하게: "本文" (나), "该模型" (해당 모델), "相关研究" (관련 연구)
- "其" 를 문장 주어로 사용 금지 (대신 명사 직접 사용)
- 방어적 글쓰기: 기본 잘 알려진 사실도 "一定程度上", "相对" 등 hedging 없이 서술
  - (단, 박사 피드백 규칙에서는 새로운 주장에 hedging 필요 → 절충 필요)
- 문장 간결성: "逐步不再...并尝试..." 같은 이중부정적 구조 사용 지양
- "进行+동词" 구조 지양: "进行结合" → "结合" 직접 사용

### **7. 단어 선택 기준 (BIT 스타일)**
- 기술 등장 서술: "被引入", "被提出", "得到广泛的应用"
- 기술 발전 서술: "随着...的发展", "在...的基础上"
- 성능 비교: "取得了更好的表现", "超过了所有的基线模型"
- 단점 서술: "缺陷在于", "存在...的问题"
- 결론 도출: "综上所述，[결론]，这为本文...提供了可行性"
